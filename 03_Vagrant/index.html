    03. Vagrant

    Vagrant is a tool for building and managing virtual machine environments in a single workflow. With an easy-to-use workflow and focus on automation, Vagrant lowers development environment setup time, increases production parity, and makes the "works on my machine" excuse a relic of the past.

    Installing

    Before we can get started using Vagrant, we need to first install it. So, please follow the instructions found here based on your OS.

    Installing - Hypervisors

    When using Vagrant (like we saw in Packer), you may at some point run into issues when you have more than one Hypervisor installed.

    Boxes

    Adding

    For our learning, we will be adding the box we built using Packer back in 02. Packer. So, let's go ahead and do this now.

    NOTE: Replace the 1595459323 with whatever matches your build.

    vagrant box add local/ubuntu1804 ubuntu1804-virtualbox-1595459323.box

    ==> box: Box file was not detected as metadata. Adding it directly...
==> box: Adding box 'local/ubuntu1804' (v0) for provider:
    box: Unpacking necessary files from: file:///Users/larrysmithjr/Git_Projects/Personal/GitHub/mrlesmithjr/hashi-learning/learning/Packer/ubuntu1804-virtualbox-1595459323.box
==> box: Successfully added box 'local/ubuntu1804' (v0) for 'virtualbox'!

    We've successfully added our first Vagrant box! And as we mentioned previously, we will be using this image for the remainder of this learning.

    However, before we move on, let's add another box.

    vagrant box add mrlesmithjr/centos7

    Listing

    Now that we've added our first box, we can get a quick list of boxes currently available by executing:

    vagrant box list

    ▶ vagrant box list
local/ubuntu1804    (virtualbox, 0)
mrlesmithjr/centos7 (libvirt, 1588339341)
